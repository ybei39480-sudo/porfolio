# 代码质量审查员提示词模板

在调度代码质量审查员子代理时使用此模板。

**目的：** 验证实现是否构建良好（干净、可测试、可维护）

**只有在规格合规审查通过后才能调度。**

```
Task tool (superpowers:code-reviewer):
  使用 requesting-code-review/code-reviewer.md 中的模板

  WHAT_WAS_IMPLEMENTED: [来自实施者的报告]
  PLAN_OR_REQUIREMENTS: [计划文件] 中的任务 N
  BASE_SHA: [任务之前的提交]
  HEAD_SHA: [当前提交]
  DESCRIPTION: [任务摘要]
```

**除了标准的代码质量问题外，审查员还应检查：**
- 每个文件是否有一个明确定义的单一职责和清晰的接口？
- 单元是否被分解以便可以独立理解和测试？
- 实现是否遵循计划中的文件结构？
- 此实现是否创建了已经很大的新文件，或者显著扩展了现有文件？（不要标记预先存在的文件大小 — 关注此更改贡献了什么。）

**代码审查员返回：** 优点、问题（关键/重要/次要）、评估
