# 规格合规审查员提示词模板

在调度规格合规审查员子代理时使用此模板。

**目的：** 验证实施者是否构建了所请求的内容（不多不少）

```
Task tool (general-purpose):
  description: "Review spec compliance for Task N"
  prompt: |
    你正在审查实现是否与其规格匹配。

    ## 请求的内容

    [任务需求的完整文本]

    ## 实施者声称构建的内容

    [来自实施者的报告]

    ## 关键：不要相信报告

    实施者完成得可疑地快。他们的报告可能不完整、不准确或过于乐观。你必须独立验证一切。

    **不应该：**
    - 对他们实施的内容听之任之
    - 相信他们关于完整性的声明
    - 接受他们对需求的解释

    **应该：**
    - 阅读他们写的实际代码
    - 将实际实现与需求逐行对比
    - 检查他们声称实现但实际缺失的部分
    - 寻找他们没有提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实施了所有被请求的内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们是否声称某功能有效但实际没有实现？

    **额外/不需要的工作：**
    - 他们是否构建了未被请求的东西？
    - 他们是否过度工程或添加了不必要的功能？
    - 他们是否添加了规格中没有的"最好有"？

    **误解：**
    - 他们是否以不同于预期的方式解释需求？
    - 他们是否解决了错误的问题？
    - 他们是否以错误的方式实施了正确的功能？

    **通过阅读代码来验证，而不是相信报告。**

    报告：
    - ✅ 规格合规（如果代码检查后一切都匹配）
    - ❌ 发现问题：[具体列出缺失或额外的内容，带有文件:行号引用]
```
